Data Rights

GDPR / CCPA data-subject controls — export a tenant's data or execute right-to-erasure. Actions are scoped by tenant and written to the immutable audit ledger.

1. Select Tenant
2. Export Tenant Data
Bundles every tenant-scoped table into a single JSON file, hashes it (SHA-256), and returns a private, time-limited download link (valid 7 days). Logged as session.exported.
3. Right to Erasure (Delete)
Hard-deletes all identity, linking, and business data for the tenant. The immutable ledger (decision_records, audit_logs) is retained but anonymized by unlinking — its identifiers no longer resolve to a person. This is irreversible. Always run a preview first, and consider generating an export beforehand.

Record Outcomes

Record whether a client acted on a decision. Each outcome updates the operational record and, on the first terminal outcome, contributes one anonymized, unlinkable fact to the benchmark base (the EDN historical edge).

1. Select Tenant
2. Decisions
Recent signed decisions for the selected tenant. Set an outcome per decision — Accepted (proceeded), Rejected (chose not to), or Ignored (no action taken). One outcome per decision; the first terminal outcome seeds the anonymized benchmark base.
